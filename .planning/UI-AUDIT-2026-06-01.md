## Pillar 1: Color Discipline (3/4)

### ✅ Strengths
- All semantic surfaces use CSS variables (`--bg`, `--surface`, `--fg1`, `--fg2`, `--fg3`)
- Status dots use muted stone/brass/green only — no traffic-light rainbow
- Brass (`--brass-*`) reserved exclusively for evidence strength
- Brick (`--brick-*`) reserved exclusively for corrections/rejections
- No neon, no saturated UI colors

### ❌ Issues

#### CRIT-01: Missing token `--shadow-lg`
**Location:** `src/components/hlshajara.css:738` (`.modal-panel`)
**Impact:** Modal panels render with no shadow (falls back to none).
**Fix:** Add `--shadow-lg: 0 4px 12px rgba(27,26,22,0.1), 0 16px 48px rgba(27,26,22,0.08);` to `tokens.css`.

#### CRIT-02: Missing token `--stone-50`
**Location:** `src/components/hlshajara.css:608` (`.skeleton` gradient)
**Impact:** Skeleton shimmer gradient has a broken stop — renders as transparent/invalid.
**Fix:** Add `--stone-50: #F7F5EE;` to `tokens.css`.

#### CRIT-03: Invalid CSS variable `--accent`
**Location:** `src/app/[locale]/reviewer/ReviewerClient.tsx:199`
**Impact:** Source link in reviewer modal has no color (inherits default).
**Fix:** Replace `var(--accent)` with `var(--brand)`.

#### MED-01: Hero radial gradient
**Location:** `src/components/hlshajara.css:348` (`.hero::before`)
**Spec violation:** "No gradients." The hero has a subtle radial gradient using `rgba(38,77,46,0.05)`.
**Note:** Extremely subtle; may be acceptable as atmospheric depth. Recommend removing to be spec-pure.

## Pillar 3: Spacing & Layout (3/4)

### ✅ Strengths
- Logical properties everywhere: `margin-inline`, `padding-inline`, `border-inline-end`
- 4px base scale (`--space-1` = 4px)
- Max-width 1080px, centered (`page-container`)
- Narrow variant 720px (`page-container-narrow`)
- Responsive breakpoints: 860px (filters), 560px (header compact), 480px (hero buttons)

**Count:** 131 `style={{...}}` occurrences across 18 files

## Pillar 4: Component Consistency (2/4)

### ✅ Strengths
- `.card`, `.btn`, `.ev`, `.status`, `.mark`, `.cite`, `.legal` all follow token system
- Form sections (`.form-section`, `.form-section-title`, `.form-row`) are consistent
- Modal system (`.modal-overlay`, `.modal-panel`, `.modal-header`, `.modal-close`) works
- Login card (`.login-card`), reviewer cards (`.reviewer-card`), dash cards (`.dash-card`) all use tokens

### ❌ Issues

#### HIGH-02: Hardcoded "Tier" not bilingual
**Locations:**
- `src/components/SourceCitation.tsx:7` — `"Tier {source.tier}"`
- `src/components/EvidenceCard.tsx:52` — `"Tier ${s.tier}"`
- `src/app/[locale]/entity/[id]/EntityDetailClient.tsx:129` — `"Tier {s.tier}"`
- `src/app/[locale]/reviewer/ReviewerClient.tsx:224-226` — `"Tier A/B/C"`

**Impact:** Arabic UI shows English "Tier" label.
**Fix:** Add `TIER_LABELS` to `src/lib/labels.ts` (AR: «المستوى أ/ب/ج» or similar).

#### MED-03: Record search input overrides `.ds-input`
**Location:** `src/app/[locale]/record/RecordClient.tsx:69`
**Issue:** `style={{ fontSize: 15, padding: "12px 16px" }}` on a `.ds-input` — defeats the token.
**Fix:** Use `.archive-search` class (already defined in CSS) or extend `.ds-input`.

#### MED-04: Empty states use ad-hoc inline padding
**Locations:** Multiple files use `style={{ padding: 40/48, textAlign: "center" }}` for empty-state cards.
**Fix:** Create `.empty-state` utility class.

#### LOW-01: EvidenceCard StatusBadge wrapper
**Location:** `src/components/EvidenceCard.tsx:43-45`
**Issue:** `<div style={{ marginBottom: 10 }}>` around StatusBadge.
**Fix:** Add `.card .status-wrap` class or use `margin-block-end` utility.

## Pillar 5: Motion & Interaction (3/4)

### ✅ Strengths
- `--dur: 200ms` with `cubic-bezier(.2,0,.2,1)` — calm, no bounce
- Cards lift `-2px` on hover with shadow deepen
- Buttons have hover (`→800`), active (`→900` + 1px nudge), disabled states
- Focus-visible: 2px brass outline, 2px offset
- Header sticky with `backdrop-filter: blur(8px)`

### ❌ Issues

#### MED-05: Native checkbox focus not visible
**Location:** `ReviewerClient.tsx`, `SubmitClient.tsx`
**Issue:** Native `<input type="checkbox">` elements lack custom focus styling. The browser default may be invisible against the warm paper background.
**Fix:** Add CSS rule: `input[type="checkbox"]:focus-visible { outline: 2px solid var(--focus-ring); outline-offset: 2px; }`

#### MED-06: File input missing cursor:pointer
**Location:** `src/app/[locale]/submit/SubmitClient.tsx:292`
**Issue:** `style={{ cursor: uploadingFile ? "not-allowed" : "pointer" }}` is inline; disabled state handled but native file input cursor is inconsistent.
**Fix:** Move to CSS class.

#### LOW-02: Skeleton animation ignores reduced-motion
**Location:** `src/components/hlshajara.css:610-616`
**Issue:** `@keyframes skeleton-shimmer` runs unconditionally.
**Fix:** Wrap in `@media (prefers-reduced-motion: no-preference) { ... }`

#### LOW-03: Login button opacity/cursor inline
**Location:** `src/app/[locale]/login/LoginClient.tsx:146-150`
**Issue:** `style={{ opacity: loading ? 0.7 : 1, cursor: loading ? "not-allowed" : "pointer" }}`
**Fix:** Use `.btn:disabled` already handles this — remove inline style.
