<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| uploaded video file → object store | Untrusted media carrying location/identifying container + stream metadata crosses into storage. |
| ffmpeg CLI invocation | A system binary is shelled out to with a tmp file path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-08 | Information disclosure | video stored with GPS/device metadata (de-anonymizes submitter/victim) | mitigate | `ffmpeg -map_metadata -1` strips all metadata before hashing/storage; if ffmpeg is absent the upload is rejected (503) rather than stored unstripped (fail closed). |
| T-33-09 | Denial of service | very large video re-encoded / unbounded ffmpeg run | mitigate | `-c copy` avoids re-encode (metadata-only, fast); execAsync timeout bounds the call; existing MAX_SIZE cap retained. |
| T-33-10 | Tampering | shell injection via crafted filename in the ffmpeg command | mitigate | Tmp file path is a server-generated random name + sanitized ext (not the client filename); no user string interpolated into the command beyond the controlled tmp path. |
| T-33-SC | Tampering | npm/pip/cargo installs | mitigate | No new npm package (no fluent-ffmpeg/ffmpeg-static); thin Node-built-in exec helper, mirroring clamav.ts; ffmpeg is a documented system dependency. |
</threat_model>
